Question 23


Assume that array arr has been defined and initialized with the values {5, 4, 3, 2, 1}. What are the values in array arr after two passes of the for loop (i.e., when j = 2 at the point indicated by / * end of for loop * / ) ?

Insertion sort works by taking an element and moving it until it is behind an element that is greater than it.
By the end of the first iteration, the element with an index of 1 (4) is moved to index 0 -> {4, 5, 3, 2, 1}
By the end of the second iteration, the element with an idex of 2 (3) is moved to the index of 0 -> {3, 4, 5, 2, 1}
At this point, j = 2 at the spot indicated, therefore the answer is {3, 4, 5, 2, 1}

Question 26


What is printed as a result of the call start() ?

Within the method start(), the method changeIt() is called. changeIt() is supposed to modify inputted variables, but it effectively does nothing since, when variables are passed into Java methods, copies are passed, not references to their location in memory. Thus, no change is made, and 1 2 3 4 5 6 blackboard is printed out, making that the correct answer.

Question 36


Suppose the binarySearch method is called with an array containing 2,000 elements sorted in increasing order. What is the maximum number of times that the statement indicated by / * Calculate midpoint * / could execute?

The binary search algorithm repeatedly splits a given array in halves until the number it searches for is in the middle of one of the halves. At its slowest, the passed array will be split into halves until the array is at a length of 1. To find the maximum number of iterations, we would need to find how many times the array can be split until it cannot be split any further. Since the array is being halves, taking the log of the size of the array with a base of 2 would give us our answer.
log2(2000) is approximately 11 (211 = 2048), so 11 is our answer.
